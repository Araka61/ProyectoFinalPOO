BOLSA EMPLEO
--------------------------------------------------------------------------------

RESUMEN GENERAL
La clase `BolsaEmpleo` es el "cerebro" de la aplicación. Centraliza toda la 
información y la lógica de negocio. Utiliza el patrón de diseńo "Singleton" 
(a través del método getInstancia) para garantizar que toda la aplicación trabaje 
con las mismas listas de datos en la memoria(lo que explico Alonso pero mas bonito).

ESTRUCTURA DE DATOS
* Colecciones: Utiliza `ArrayList` para almacenar todas las entidades del 
  sistema (Persona, Empresa, Oferta, Solicitud y Usuario).
* Generadores de ID: Maneja variables estáticas enteras (ej. `generadorIdPersona`) 
  que actúan como contadores autoincrementables al registrar nuevas entidades.
* Despues cuando se implementen ficheros se deve de cargar desde el fichero los generadores de ID 
  para evitar duplicar IDs

MÓDULOS Y FUNCIONALIDADES

1. REGISTRO (Altas)
* registrarPersona / registrarEmpresa / registrarSolicitud: Ańaden el objeto a 
  su lista correspondiente y aumentan el generador de ID en 1.
* registrarOferta: Además de guardarla en la lista general, se encarga de 
  vincular la oferta directamente a la empresa que la publica.
*registrarPersona tambien registra un usurio, toda persona en el sistema deve de 
 ser dependiente de un usuario de forma de que puedan compartir ID y estar relacionados
 sin que uno contenga al otro

2. BÚSQUEDA Y FILTRADO
* Búsquedas por ID: Métodos clásicos (`buscarPersona`, `buscarEmpresa`, etc.) 
  que recorren las listas usando un ciclo `while` y devuelven el objeto si el ID 
  coincide (ignorando mayúsculas/minúsculas)(sacados desde LaGiraldilla, nada nuevo o del otro mundo).

* Filtros específicos: 
  - `getPersonasDisponibles`: Devuelve solo a las personas que no están empleadas.
  - `getPersonasPorNivel`: Filtra a las personas usando `instanceof` para saber 
    si son de tipo Técnico, Grado o Trabajador.
  - `getUsuarioPorUserName`: Busca un usuario específico por su nombre de usuario.

3. ALGORITMO DE MATCHING
* `PorcentajeCoincidencia`: Toma una oferta de la empresa y la compara contra 
  todas las solicitudes activas de personas que actualmente no están empleadas. 
  Si los puntos de coincidencia superan el mínimo exigido por la oferta, guarda 
  el ID del candidato.
* El puntaje se calcula en bloques de 10 puntos evaluando:
  - Disponibilidad y tipo de jornada.
  - Experiencia requerida vs. Rango salarial esperado.
  - Preferencias personales (sexo, licencia de conducir, disponibilidad para mudarse).
  - Ubicación/Residencia.

4. VALIDACIONES Y COMPROBACIONES
* `login`: Valida las credenciales. Busca al usuario y, si existe, verifica que 
  la contraseńa coincida exactamente.
* `aptitudSolicitud`: Metodo por el momento incompleto, el objetivo es determinar el nivel necesario
  para el puesto aplicado 
* `existeUsuario`: en el futuro tendremos que tener un registro al usuario estar vinculado a una contrasena
  no se puede repetir un UserName lo verifique y es lo mas comun.

RESUMEN GENERAL

Las clases Login y MenuPrincipal manejan el arranque y la seguridad inicial
de la aplicación. MenuPrincipal es el marco principal (JFrame) y Login actúa
como una ventana (JDialog {Como deve de ser}) que bloquea el sistema hasta que el usuario se
autentique (método login de la clase controladora). Trabajan directamente con BolsaEmpleo para validar datos y con
GestorFicheros para el tema de los datos guardados.

ESTRUCTURA DE DATOS

Componentes: Usa JTextField, JPasswordField, JButton y JRadioButton
para la interacción del usuario.

Eventos: Utiliza keyReleased para validar cosas en tiempo real
mientras se escribe (basicamente detectar cuando se escribe y cuando se pulsa un boton).

No guarda listas propias, solo interactúa con el Singleton y manda a llamar métodos
de las clases lógicas.

MÓDULOS Y FUNCIONALIDADES

ARRANQUE (Main y MenuPrincipal)

Cargar Datos: El main arranca llamando a GestorFicheros.cargarDatosDesdeFicheros()
para tener toda la info lista antes de hacer cualquier cosa.

Control de Sesión (Cookies): Revisa si hay un usuario guardado (getCookieUsuario()).
Si lo hay, abre el menú de una; si no, lanza el Login.

AUTENTICACIÓN (Login)

btnLogin: Empiesa desactivado hasta que se llena usuario y password, llama a BolsaEmpleo.getInstancia().login(...).
Si entra, guarda las cookies, muestra el menú y se cierra. Si falla, tira un JOptionPane de advertencia.

Mostrar contraseńa: Un radio button que cambia el EchoChar a (char)0 para que
se vea el texto y lo vuelve a poner en '*' si se desmarca.

VALIDACIONES

validarCampos: Verifica cada vez que sueltas una tecla (keyReleased) si los
campos de usuario y contraseńa tienen algo. Si están llenos, habilita el botón y
lo pone verde. Si falta uno, lo bloquea y lo pone rojo.
